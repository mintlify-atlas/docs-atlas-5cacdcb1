OpenBeken (originally OpenBK7231T) is an open-source firmware project that frees Tuya-based IoT hardware from vendor lock-in. Where the stock firmware calls home to Tuya's cloud and cannot be locally controlled, OpenBeken gives you a Tasmota-compatible interface with full MQTT support, Home Assistant auto-discovery, an advanced scripting engine, and a web-based configuration UI — all running entirely on your local network. Although the project started targeting Beken's BK7231T chip, it has grown into a multi-vendor platform supporting chips from Beken, Bouffalo Lab, Espressif, WinnerMicro, XRadio, Realtek, Lightning Semi, ESWIN, and more.

## Device template database

OpenBeken maintains a community-driven device template database with more than 800 entries. When you open the web application on a freshly flashed device, you can search the database and apply a template that pre-configures all GPIO roles for your specific hardware — no manual pin mapping required. The database is also accessible directly from the firmware without leaving the device's web UI.

## Tasmota compatibility

<Note>
  OpenBeken supports the common Tasmota JSON command format over HTTP and MQTT, including TELE, STAT, and CMND topic namespaces. Existing Tasmota rules, ioBroker flows, and Home Assistant MQTT configurations generally work without modification.
</Note>

## Supported platforms overview

OpenBeken runs on hardware from at least eight silicon vendors. See the [supported platforms](/supported-platforms) page for the full feature matrix covering GPIO, OTA, PWM, ADC, deep sleep, SPI LED, IR, and Bluetooth support across every chip.

## Building from source

Pre-built binaries for every supported platform are published on the GitHub Releases tab. If you need a custom build — for example to enable Berry scripting or adjust partition layout — see the [building guide](/advanced/building) for Docker and manual build instructions.
