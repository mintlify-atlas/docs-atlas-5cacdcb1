OpenBeken exposes up to 32 numbered **channels**, each of which is simply an integer variable. A channel has no inherent meaning on its own тАФ a relay driver reads channel 1 and closes its output when the value is `1`; a PWM driver reads channel 3 and sets its duty cycle accordingly. What a channel **represents** to a human тАФ whether it is an on/off toggle, a percentage dimmer, a temperature in tenths of a degree, or a power reading in watts тАФ is determined by its **channel type**.

Channel types serve two purposes: they control the UI widget shown on the main web panel for that channel (toggle button, slider, text field, read-only label, radio buttons), and they instruct the firmware how to scale or format the raw integer value before displaying it. For example, a TuyaMCU device that sends temperature as an integer multiplied by 10 (so 254 means 25.4 ┬░C) needs the `Temperature_div10` channel type so the display shows `25.4` instead of `254`.

## Setting channel types

Open **Config тЖТ Channel config** in the web interface. Each channel row shows its current type in a dropdown. Select the type that matches what the channel is used for and click **Save**. The main page updates immediately to show the new widget.

<Note>
You only need to set channel types for channels that are actually in use. Unused channels with the **Default** type are not displayed on the main page.
</Note>

## General-purpose types

<Tabs>
  <Tab title="Toggle and relay">
    | Type | Description |
    |------|-------------|
    | `Default` | The standard on/off channel for relays and simple switches. A **Default** channel with a non-zero value shows as ON; zero shows as OFF. This is the correct type for almost all relay and switch channels on non-TuyaMCU devices. |
    | `Toggle` | Like Default, but the toggle widget is always shown even if the channel is currently zero. Useful for channels you always want to be visible and interactive. |
    | `Toggle_Inv` | Like Toggle but the displayed ON/OFF state is inverted тАФ the widget shows ON when the channel value is `0`. |
    | `ReadOnly` | Displays the raw integer value as a read-only label. You can still write to the channel from scripts or console commands; it just cannot be changed from the web panel UI. |
  </Tab>

  <Tab title="Dimmers and sliders">
    | Type | Description |
    |------|-------------|
    | `Dimmer` | Shows a 0тАУ100 range slider. The slider value is written to the channel. Use this for TuyaMCU dimmers, or for anything you want to map to a 0тАУ100 percent scale. |
    | `Dimmer256` | Like Dimmer but uses a 0тАУ255 range. Suitable for 8-bit PWM dimmers accessed through TuyaMCU. |
    | `Dimmer1000` | Like Dimmer but uses a 0тАУ1000 range, matching the native PWM resolution used by OpenBeken's LED system. |
    | `OffDimBright` | A three-option radio button for lighting: Off, Dim, and Bright. |

  <Tab title="Multi-state radio buttons">

    | `OffLowMidHighHighest` | Five-state radio (0тАУ4). |
    | `OffLowestLowMidHighHighest` | Six-state radio (0тАУ5). |
    | `LowestLowMidHighHighest` | Five-state radio without an explicit Off state. |
    | `ReadOnlyLowMidHigh` | Read-only version of LowMidHigh тАФ shows the label but does not allow changes from the web panel. |

## Sensor channel types

Use these types when a pin role (DHT22, SHT3X, CHT83XX, etc.) writes sensor readings to a channel, or when TuyaMCU forwards sensor data from an external MCU.

| Type | Display |
|------|---------|
| `Temperature` | Integer temperature in degrees Celsius, read-only. |
| `Temperature_div10` | Temperature from TuyaMCU тАФ raw value divided by 10. `254` displays as `25.4 ┬░C`. |
| `Temperature_div100` | Temperature divided by 100. |
| `Temperature_div2` | Temperature multiplied by 0.5 (divide-by-2 equivalent). |
| `Humidity` | Integer humidity in percent, read-only. |
| `Humidity_div10` | Humidity from TuyaMCU тАФ raw value divided by 10. `554` displays as `55.4 %`. If you want the multiplied raw value published to MQTT instead of the divided display value, enable the relevant MQTT flag in **General options**. |
| `Illuminance` | Illuminance in lux. |
| `Illuminance_div10` | Illuminance divided by 10 before display. |
| `SmokePercent` | Smoke concentration as a percentage. |
| `OpenClosed` | Displays `Open` or `Closed` based on channel value. Suitable for door and window sensors. |
| `OpenClosed_Inv` | Like OpenClosed with values inverted. |
| `BatteryLevelPercent` | Displays the value as a battery level percentage. |
| `Motion` | Motion presence indicator. |
| `Motion_n` | Motion presence with inverted logic. |
| `Ph` | pH water quality reading. |
| `Orp` | ORP water quality reading. |
| `Tds` | TDS water quality reading. |
| `Pressure_div100` | Atmospheric pressure in hPa, raw value divided by 100. |

## TuyaMCU power metering types

When a TuyaMCU device (or an external MCU) sends power-metering data, it transmits raw integers because the TuyaMCU protocol carries no floating-point values. The following channel types apply the correct divisors so that readings display correctly on the web panel and are published with the right units over MQTT.

<Note>
These types are **not** used for direct hardware power-metering chips such as BL0942, BL0937, or CSE7766. Those chips have their own drivers that handle scaling internally. Use these types only for TuyaMCU data point channels.
</Note>

| Type | Divisor | Typical use |
|------|---------|-------------|
| `Voltage_div10` | ├╖10 | Mains voltage тАФ TuyaMCU sends `2310` for 231.0 V |
| `Voltage_div100` | ├╖100 | Mains voltage тАФ TuyaMCU sends `23100` for 231.00 V |
| `Current_div100` | ├╖100 | Current in amps тАФ TuyaMCU sends `150` for 1.50 A |
| `Current_div1000` | ├╖1000 | Current in amps тАФ TuyaMCU sends `1500` for 1.500 A |
| `Current_div10` | ├╖10 | Current in amps |
| `Power` | ├Ч1 | Active power in watts, integer |
| `Power_div10` | ├╖10 | Power with one decimal place |
| `Power_div100` | ├╖100 | Power with two decimal places |
| `ActivePower` | ├Ч1 | Active power (alternative label) |
| `ReactivePower` | ├Ч1 | Reactive power in VAr |
| `PowerFactor_div1000` | ├╖1000 | Power factor тАФ TuyaMCU sends `950` for 0.950 |
| `PowerFactor_div100` | ├╖100 | Power factor |
| `Frequency` | ├Ч1 | AC frequency in Hz |
| `Frequency_div10` | ├╖10 | Frequency with one decimal place |
| `Frequency_div100` | ├╖100 | Frequency тАФ TuyaMCU sends `5000` for 50.00 Hz |
| `Frequency_div1000` | ├╖1000 | Frequency with three decimal places |
| `EnergyTotal_kWh_div1000` | ├╖1000 | Total energy in kWh тАФ TuyaMCU sends `1234` for 1.234 kWh |
| `EnergyTotal_kWh_div100` | ├╖100 | Total energy in kWh |
| `EnergyToday_kWh_div1000` | ├╖1000 | Today's energy in kWh |
| `EnergyExport_kWh_div1000` | ├╖1000 | Exported energy in kWh |
| `EnergyImport_kWh_div1000` | ├╖1000 | Imported energy in kWh |
| `LeakageCurrent_div1000` | ├╖1000 | Leakage current in mA |

## Custom enum types

Two special channel types allow you to define a mapping between integer values and human-readable string labels:

**`Enum`** тАФ Creates a selectable dropdown on the web panel. Use the `SetChannelEnum` command to define the label mapping. This is ideal for TuyaMCU data points that represent modes (for example: `0=Normal`, `1=Eco`, `2=Turbo`).

**`ReadOnlyEnum`** тАФ Like Enum but the channel is read-only in the web panel; the value can still be updated by TuyaMCU or scripts.

```bash
# Example: define labels for channel 4 (a three-mode fan speed)
SetChannelEnum 4 0=Off 1=Low 2=High
```
