OpenBeken supports more than 30 chipsets from eight silicon vendors. What began as firmware for Beken's BK7231T has grown into a multi-vendor platform spanning Beken, Bouffalo Lab, WinnerMicro, XRadio/Allwinner, Lightning Semi, Espressif, Realtek, ESWIN, RDA, and Taixin. Each platform shares the same web interface, command system, scripting engine, and MQTT integration вАФ only the low-level peripheral drivers differ. The table below shows which hardware features are available on each chip so you can assess compatibility before purchasing or flashing a device.

## In-progress platforms

Support for the following chips is actively under development and not yet in stable releases:

- **XR872 / XF16** вАФ found in A9 camera modules
- **BK7252U** вАФ found in Tuya doorbell and camera hardware
- **BK7252N** вАФ found in A9 camera PCBs

## Choosing a chip for new projects

If you are selecting hardware for a new project, BK7231N (via Tuya modules CB2S or CB2L) offers the broadest feature coverage including Bluetooth, WPA3 (in `_ALT` builds), full OTA, and SPI LED support. For ESP-ecosystem compatibility or if you need strong toolchain support, any ESP32 variant is a solid choice. Avoid XR809 if OTA updates are important to your workflow.

  Pre-built firmware binaries for every supported platform are published on the [GitHub Releases](https://github.com/openshwprojects/OpenBK7231T_App/releases) page. If you need to build for a platform not yet in the release pipeline, see the [building guide](/advanced/building).
